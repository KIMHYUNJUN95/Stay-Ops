# Admin Todoist Console (Dashboard) — Planning Spec

Status: **Planning (2026-07-24).** 이 문서는 대시보드(어드민 웹) Todoist의 기획/IA/동작 스펙이다.
디자인(시각)은 대표님이 직접 진행하며, 이 문서를 기준으로 한다. 구현은 아직 시작하지 않았다.

## 1. Purpose & Scope (심플 원칙)

대시보드 Todoist는 **모바일 Todoist의 본래 목적을 큰 화면에서 그대로 쓰는 것**이 전부다. 여기에 사무실
관리자를 위한 **업무 지시(Work Directive)** 하나만 얹는다.

**In scope**
- 모바일 Todoist **코어 기능 전부**(생성/편집/완료/공유/반복/기간/태그/사진/컨텍스트 링크/캘린더/프로젝트)를
  데스크톱 레이아웃으로.
- **업무 지시** — 사무실 관리자가 대시보드에서 특정 멤버에게 "오늘 이 업무 하라"고 기록해 주는 기능
  (§7). 기존 **공유(share) 모델 재사용** + 받는 사람 쪽 "지시" 표시.

**Out of scope (일부러 넣지 않음 — "번거롭지도 무겁지도 않게")**
- 팀 전체 조망/감사 대시보드, 멤버별 워크로드/완료율 KPI, 업무일지 취합 등 **관리자 분석/오버사이트 레이어**.
- 저장된 필터·라벨, 미리 알림, 전역 검색 등 **부가 기능**.
- **정식 담당자(assignee) 필드** — §8(향후 확장)으로 보류.

> 대시보드 Todoist는 "관리 콘솔"이 아니라 **"큰 화면 Todoist + 사무실 업무 지시"** 다. 다른 어드민
> 모듈(주문·분실물·수리·공지)이 "관리 느낌"인 것과 달리, Todoist는 **기능이 모바일과 같아야** 한다.

## 2. Locked Decisions (2026-07-24 기획 회의)

| 항목 | 결정 |
| --- | --- |
| 성격 | 모바일 투두 코어 그대로 + 업무 지시 1개. 관리자 분석/오버사이트/부가기능은 **미포함**. |
| 사용자 | 어드민 웹 접근 = 정직원(owner / senior_managing_director / office_admin / cs_staff / field_manager / staff). **파트타이머는 모바일 전용**(웹 미접근, 기능/데이터는 동일). |

## 3. Layout & IA (Todoist Desktop)

벤치마크: Todoist 데스크톱 앱. `/admin/*` 공용 콘솔의 "관리형" 패턴이 **아니라**, Todoist 본연의
개인 작업 워크스페이스를 따른다.

```
┌───────────────┬───────────────────────────────────────────┐
│  Sidebar       │  Main pane                                 │
│  ─────────     │  ───────────                               │
│  ＋ 작업 추가   │  [뷰 제목: 오늘 · N작업]                    │
│  🔎 검색       │  ◯ 작업 …                        (관리함) │
│  📥 관리함  N   │  ◯ 작업 …                        (관리함) │
│  📅 오늘   N   │  ┌ 인라인 작업추가 카드 ─────────────────┐ │
│  ▷ 내일        │  │ 작업 이름 / 설명                        │ │
│  🗂 프로젝트    │  │ [📅 날짜] [🏳 우선순위] [👤 대상] [＋]  │ │
│  📤 공유함      │  │                     [취소] [작업 추가]  │ │
│  ✓ 완료·기록   │  └─────────────────────────────────────────┘ │
│  🗓 캘린더      │                                            │
│  ───────       │                                            │
│  🗂 프로젝트     │      (작업 클릭 → 우측 상세 패널/모달)      │
│   # Haru-Ops    │                                            │
│   # PMS         │                                            │
└───────────────┴───────────────────────────────────────────┘
```

- **사이드바** — ＋작업추가(상단), 검색, 뷰 목록(관리함·오늘·내일·공유함·완료·캘린더 + 카운트 배지),
  프로젝트 목록(멤버 있는 공유 프로젝트). 하단 계정.
- **메인** — 선택한 뷰의 작업 목록 + **인라인 작업추가 카드**(Todoist처럼 목록 하단/상단에서 바로 입력).
- **우측 상세** — 작업 클릭 시 상세 패널(또는 모달). 모바일 상세와 동일 정보(본문/일정/기간/반복/태그/
  사진/컨텍스트/공유·참여자/업데이트 로그).
- 반응형: 좁은 폭에선 사이드바 접힘(햄버거). 데스크톱 우선.

> 공용 어드민 콘솔 계약(§4 admin-console.css: opsbar/qtbl/panel 등)을 **강제 적용하지 않는다.**
> Todoist 워크스페이스는 별도 시각 언어를 갖는다(디자인은 대표님이 진행).

## 4. Views (모바일 패리티)

모바일 7뷰를 그대로 데스크톱에. 각 뷰의 정의·정렬·필터 규칙은 **모바일과 동일**(18 문서 기준).

| 뷰 | 내용 | 비고 |
| --- | --- | --- |
| 오늘 | 지연(overdue) + 오늘 기준일 작업 | 지연 섹션 상단, 지난-작업 일정변경/삭제 배너 포함(모바일과 동일) |
| 내일 | 내일 기준일 작업 | |
| 관리함 | 활성 작업 전체(관리) | 빠른추가 기본 도착지 |
| 프로젝트 | 프로젝트+섹션별 작업 | 프로젝트 작업은 이 뷰에만 |
| 공유함 | 내가 공유/지시한 작업 | **업무 지시한 작업이 여기 모임**(§7) |
| 완료·기록 | 완료일별 그룹 | 업무일지(보고서) 버튼 — 모바일과 동일(각자 본인 것) |
| 캘린더 | 월/아젠다 | 반복은 가상 미리보기 |

- 뷰 전환은 사이드바. 목록/정렬/그룹핑/검색·날짜필터 규칙은 모바일 `TasksWorkspace`와 동일 규칙을
  따른다(중복 로직이 아니라 **동일 데이터·동일 정의** 재현).
- 데스크톱 편의: 넓은 화면이므로 목록 + 우측 상세를 **동시에** 볼 수 있다(모바일은 화면 전환).

## 5. Create / Edit (인라인 추가 + 업무 지시)

- **인라인 작업추가** — Todoist처럼 메인에서 바로 입력(작업 이름 → 엔터로 추가). 확장 시 설명/날짜/
  우선순위/**대상(지시)** 칩. "작업 추가"로 저장.
- **일정 피커** — 모바일과 **동일한 통합 일정 피커**(단일 날짜 + 빠른옵션 오늘/내일/다음 주/다음 주말/
  날짜 없음 + 달력 + 시간 + 기간(duration) + 반복(문맥형: 매일/매주 {요일}/평일마다/매월 {일}/매년 {월일})).
  데스크톱은 팝오버로 뜨는 형태(모바일은 BottomSheet) — **동작·데이터는 동일**.
- **상세 편집** — 작성자만 핵심 편집(모바일 규칙 동일). 참여자는 업데이트 로그.
- **대상(지시) 칩** — 작업 만들 때 "대상: [멤버 1명]"을 지정하면 **업무 지시**가 된다(§7). 지정 안 하면
  개인 작업(기존과 동일).

## 6. Detail Panel

모바일 상세와 **동일 정보**를 우측 패널로:
- 제목 / 상태(완료·다시 열기) / 우선순위 / 일정(날짜·시간·기간·반복) / 태그
- 본문 / 첨부 사진(뷰어) / 컨텍스트 링크(건물·객실·예약·게스트, "예약 보기")
- 공유·참여자(원작성자·첫 수신자·현재 참여자) / **업무 지시 표식**(§7)
- 통합 업데이트 로그(노트 + system 이벤트: edited/shared/completed/reopened)
- 액션: 완료/다시 열기, 편집(작성자), 공유/지시, 참여자 관리, 삭제 — 모바일과 동일 권한 규칙

## 7. Work Directive (업무 지시) — 핵심 신규

### 7.1 개념
사무실 관리자가 대시보드에서 **특정 멤버에게 "이 업무를 (오늘) 하라"고 기록**해 주는 것. 정식 담당자
개념을 새로 만들지 않고, **기존 공유(참여자) 모델을 "한 명에게 지시"로 사용**한다.

- **지시자 = 작성자(created_by)** — 매니저. 내용을 정하고(핵심 편집 권한), 완료 여부를 추적.
- **대상 = 참여자(participant)** — 그 작업을 수행. 자기 오늘/관리함에 뜬다. 완료/진행 메모 가능,
  핵심 내용은 수정 불가(지시 내용은 매니저가 고정).
- **"누가 무엇을"은 자동 확보** — 지시자는 작성자로, 내용은 작업 그 자체로 이미 데이터에 있다.

### 7.2 받는 사람이 확실히 알게 (필수)
받는 사람 쪽(모바일·대시보드 모두)에 **"[매니저 이름] 지시" 표식**을 명확히 보여준다. 이것이 일반
peer 공유와 구분되는 지점이다.

- 표식은 카드/상세에 **지시 칩**으로: 예) `👤 김철수 지시`.
- peer 공유(동료끼리 공유)와 지시(관리자→멤버)를 구분하려면 **가벼운 지시 표식 하나**가 필요하다.
  구현 옵션(무게 최소순, 구현 단계에서 확정):
  - (a) 지시 플래그 `is_directive boolean`(또는 `directed_by_user_id`) 1개 — 지시로 만든 작업만 표시.
  - (b) 별도 컬럼 없이 "작성자 role이 관리자 + 단일 참여자"로 추론 — 표식이 부정확해질 수 있어 (a) 선호.
- **이 표식은 대시보드 '업무 지시' 기능의 "받는 쪽(모바일) 절반"** 이다. 모바일 종료 상태를 다시 여는
  게 아니라, 새 기능이 태생적으로 양면(대시보드 지시 + 모바일 수신 표시)을 갖는 것.

### 7.3 동작
- 대시보드에서 작업 만들 때 **대상 1명 지정** → 그 멤버에게 공유(지시)됨 + 날짜(예: 오늘) 지정.
- 대상의 모바일/웹 목록에 즉시 뜬다(같은 DB·공유 재사용, 별도 동기화 없음).
- 매니저는 **공유함 뷰**에서 "내가 지시한 작업"을 본다(작성자=나 & 지시 표식).
- **거절 불가(강제)는 넣지 않는다** — 참여자 self-remove는 모바일 규칙 그대로. 통제는 매니저가 공유함/
  작업 상태로 확인.

### 7.4 데이터/구현 메모 (기획 관점)
- 모델 재사용: `tasks`(작성자=매니저) + `task_participants`(대상=참여자). 스키마 신규는 §7.2의 지시
  표식 컬럼 **1개 정도**가 최대(구현 시 확정). 그 외 변경 없음.
- 알림: 기존 `task_shared` 재사용(대상에게 발송). 필요 시 문구만 "지시"로.

## 8. Sync & Route

- **동기화** — 대시보드와 모바일은 **같은 `tasks`/`task_participants`/`task_updates`/`projects` 테이블,
  같은 서버 액션**을 쓴다. 한쪽에서 만들거나 완료하면 다른 쪽에 그대로 반영. 실시간 구독은 불필요(다른
  콘솔처럼 액션 후 `router.refresh()`/revalidate로 충분).
- **읽기** — `getVisibleTasks` / `getTaskDetail` / `getVisibleProjects` / `getProjectDetail` 등 기존
  세션 스코프 lib를 그대로 재사용(표면 무관).
- **쓰기** — 기존 서버 액션 재사용. 데스크톱 인플레이스 UX를 위해 필요한 경우 결과 반환형 얇은 래퍼를
  둔다(공지 콘솔에서 쓴 `{ok}` 반환 패턴). 기존 모바일 액션 동작은 건드리지 않는다.
- **라우트** — 신설 `/admin/tasks` 권장(사이드바 라벨 "Todoist"). legacy `/admin/recurring-work`는
  `/admin/tasks`로 리다이렉트. (현재 `/admin/recurring-work`는 "준비 중" 플레이스홀더.)

## 9. Mobile Parity Mapping (체크리스트)

대시보드 구현 시 아래가 모바일과 **동일 동작**인지 확인한다(문서 대조용).

- [ ] 오늘/내일/관리함/공유함/완료/캘린더/프로젝트 7뷰 + 카운트 배지
- [ ] 인라인 빠른추가(→관리함) + 상세 생성(날짜/공유/시간/기간/우선순위/태그/사진/반복/컨텍스트)
- [ ] 통합 일정 피커(단일 날짜 · 빠른옵션 · 달력 · 시간 · 기간 · 문맥형 반복+매년)
- [ ] 완료/다시 열기(상태 서클) + 완료·기록 그룹 + 업무일지(본인)
- [ ] 공유(다중 참여자) / 참여자 관리 / 원작성자 leave = 삭제 / 참여자 self-remove
- [ ] 반복 롤포워드(완료 시 다음 회차) · 지난-작업 일정변경(단일 날짜 피커) · 삭제
- [ ] 프로젝트 + 섹션 + 프로젝트 작업(사진 20장) + 멤버 초대
- [ ] 컨텍스트 링크(건물·객실·예약·게스트) + "예약 보기"
- [ ] 캘린더 월/아젠다 + 반복 가상 미리보기
- [ ] **업무 지시**(대상 지정 → 공유) + **받는 사람 "[매니저] 지시" 표식**
- [ ] i18n ko/ja/en / 권한(part-time 제외 웹 접근) / Tokyo 기준일
